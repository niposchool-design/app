# Normas do Projeto Nipo School

## 🚨 REGRAS FUNDAMENTAIS DE DESENVOLVIMENTO

### 1. Servidor de Desenvolvimento
- **NUNCA** execute `npm run dev` durante assistência
- O servidor dev SEMPRE estará rodando em um terminal bash separado
- Use sempre outro terminal para comandos, mantendo o servidor ativo
- O usuário é responsável por manter o servidor rodando

### 2. Gestão de Terminais
- Terminal 1: Servidor dev (intocável)
- Terminal 2+: Comandos de desenvolvimento, testes, git, etc.

### 3. Fluxo de Trabalho
- Assumir que o servidor está sempre ativo
- Fazer alterações nos arquivos
- Servidor irá hot-reload automaticamente
- Testar funcionalidades via navegador

### 4. Comandos Permitidos
✅ PODE usar:
- `git` comandos
- `npm install` 
- Testes unitários
- Scripts personalizados
- Análise de código

❌ NUNCA usar:
- `npm run dev`
- `npm start` 
- Qualquer comando que inicie servidor

### 5. Debugging
- Se há problemas, verificar arquivos de configuração
- Corrigir imports/exports
- Otimizar dependências
- Mas NUNCA parar/reiniciar o servidor
